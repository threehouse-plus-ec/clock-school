# Contributing to Clock School

Thank you for your interest in contributing. This project exists because people care about honest science education with minimal barriers. Contributions of all kinds are welcome.

## What We Need

- **Testing and feedback** — Try the experiments. Report what works, what confuses, what breaks.
- **Translations** — The programme is written in English but should be accessible worldwide.
- **Tier 0 field reports** — Did you run Experiment 0.1? What did you observe? Share your data.
- **Tier 1 hardware testing** — Build the kit. Document your experience. Report component substitutions that work.
- **Tier 2 notebooks** — Improve the computational exercises. Fix bugs. Add visualisations.
- **Tier 3 analysis** — Work with real datasets. Document your methods.
- **Teaching notes** — Did you use this material with a class or club? Share what you learned.

## How to Contribute

1. Fork the repository.
2. Create a branch for your contribution.
3. Make your changes.
4. Submit a pull request with a clear description of what you changed and why.

For small fixes (typos, broken links), a direct pull request is fine. For larger contributions (new experiments, major restructuring), please open an issue first to discuss.

## Standards

- **Clarity over cleverness.** Write so that a motivated teenager can follow.
- **Show your work.** Document assumptions, parameters, and reasoning.
- **Respect the invariant.** Every contribution should be consistent with the programme's organising principle: a clock is an operational comparison between periodic processes.
- **Licence compliance.** Text contributions fall under CC BY-SA 4.0. Code contributions fall under MIT. By contributing, you agree to these terms.

## Code of Conduct

Be kind. Be honest. Be constructive. Disagreement is welcome; hostility is not.

If you experience or witness behaviour that violates these principles, contact the project maintainer.

## Questions?

Open an issue or contact the project maintainer directly.
